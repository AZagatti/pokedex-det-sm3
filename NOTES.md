# Build Notes

- 2026-07-10: Handoff from a prior session claimed files were written, but the directory was empty on resume (only SPEC.md present). Starting the build fresh from scratch.
- 2026-07-10: Nested subagents are blocked in this environment (global hook `deny-nested-agents.sh`). The spec's "spawn a subagent to review each feature" step is done via top-level Agent tool calls instead of literal sub-subagents.
- 2026-07-10: SvelteKit scaffold (`sv create`, minimal+TS template) did not create a svelte.config.js — had to add it manually for adapter-static + paths.base. Newer sv versions expect adapter config in vite.config.ts's sveltekit() call OR svelte.config.js; mixing both triggers "svelte.config.js is ignored" warning. Fixed by keeping adapter/paths only in svelte.config.js and calling sveltekit() with no args in vite.config.ts.
- 2026-07-10: Repeated large Write tool calls to this session intermittently corrupted files — the file ends up literally containing the string "[trimwire: wrote ... — read the file to restore]" instead of real content. Hit this on +page.svelte (home), berries/+page.svelte, berries/[name]/+page.svelte, favorites/+page.svelte, BerryCard.svelte, playwright.config.ts, and this NOTES.md file itself (multiple times). Mitigation: after every Write, grep the file for the literal string "trimwire" and re-write if found; prefer smaller Writes + Edit for appends over large single Writes.
- 2026-07-10: PokeAPI's `/pokemon` list endpoint doesn't include types or sprites — had to fetch all 18 `/type/{name}` endpoints once at load time to build an id→types map for the grid (cheaper than N+1 per-pokemon fetches for 1351 entries).
- 2026-07-10: Zod schema bug caught by a unit test, not manually: `pokemon.cries.legacy`/`latest` can be `null` in real API responses, not just a string — schema required `.string().optional()` which rejects `null`. Fixed to `.string().nullable().optional()`. Good example of why the "write tests" step earns its keep even on an API-shape-driven app.
- 2026-07-10: `rm -rf .svelte-kit node_modules/.vite` was needed once to fix a stale-cache bug where Tailwind v4 wasn't generating the `flex-wrap` utility class (it existed in source, wasn't tree-shaken, just cached-stale). Not a real Tailwind/Svelte bug — just a dev-server cache invalidation issue after several file edits in short succession.
- 2026-07-10: Background dev server processes (`run_in_background` via Bash tool) got killed unexpectedly by the environment a few times during long tool-call sequences (no OOM/crash in logs, just vanished). Not something I could root-cause; just restarted the server each time it happened. No impact on the app itself.
- 2026-07-10: chrome-devtools MCP `navigate_page` intermittently silently dropped its `url`/`type` parameters mid-session (calls "succeeded" but did nothing, or errored with missing required field). Worked around by using `new_page` (which reliably accepted params) instead of `navigate_page` for cross-page navigation checks.
- 2026-07-10: `npx playwright install --with-deps` failed (no sudo/password in this sandbox) but the Chromium binary was already present at `~/.cache/ms-playwright` (presumably from the chrome-devtools MCP setup), so `npx playwright install chromium` (no --with-deps) was a no-op/success and e2e tests can run fine without the OS deps step.
